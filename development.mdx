Follow these steps to turn a document into structured LLM ready data in just 5 steps

<Steps>

<Step title="Install the Ingestor SDK">
    <CodeGroup>
      ```bash Python
      pip install ingestorai
      ```

      ```bash Node
      npm install ingestorai
      # or: pnpm add ingestorai
      ```
    </CodeGroup>
</Step>

<Step title="Initialize client">
    Navigate to your dashboard to obtain a copy of your API key
    <CodeGroup>
      ```bash Python
      from ingestor import Ingestor

      client = Ingestor(api_key="INGESTOR_API_KEY")
      ```

      ```bash Node
      import { Ingestor } from "ingestorai";

      const client = new Ingestor({ apiKey: "INGESTOR_API_KEY" });
      ```
    </CodeGroup>
</Step>

<Step title="Call document parsing API">

    <CodeGroup>
      ```bash Python
      # input formats: email, pdf, docx, doc, xlsx, xls, csv, xml, jpg, jpeg, png, gif, bmp, tiff, pptx, ppt
      # input can be a local path or file-like object

      response = client.content.parse(
          input="loan_application_john_smith.pdf",  
          processing_options={
              "ocr_mode": "layout_aware",
              "classify_content": ["bank_statement", "loan_application", "id_document", "other"], 
              "generate_document_title_metadescription": True,
              "split_into_documents": True, # If your file contains multiple documents (e.g. a scan of several forms).
              "chunking_strategy": "page",
              "contextualize_chunks": True, # Agentic process adds missing context            
              "generate_table_summary": True, 
              "generate_figure_summary": True
          }
      )

      job_id = response["job_id"]
      ```

      ```bash Node
      // input formats: email, pdf, docx, doc, xlsx, xls, csv, xml, jpg, jpeg, png, gif, bmp, tiff, pptx, ppt
      // input can be a local path, File/Blob (browser), or Buffer/stream

      const response = await client.content.parse({
        input: "loan_application_john_smith.pdf",
        processing_options: {
          ocr_mode: "layout_aware",
          classify_content: ["bank_statement", "loan_application", "id_document", "other"],
          generate_document_title_metadescription: true,
          split_into_documents: true,
          chunking_strategy: "page",
          contextualize_chunks: true,
          generate_table_summary: true,
          generate_figure_summary: true
        }
      });

      const job_id = response.job_id;
      ```
    </CodeGroup>
</Step>

<Step title="Poll for job completion">

    Note: As requests run through a pipeline, they can take several minutes to complete. Additional `processing_options` may increase this time.


      ```bash Node
      const documents = await client.jobs.get_document_response(job_id, { include: ["layout_bbox", "blocks"] });
      console.log(documents);
      ```

<Step title="Extract structured output (optional)">

    <CodeGroup>
      ```bash Python
    from pydantic import BaseModel, Field
    from typing import List, Optional

    class BankStatement(BaseModel):
        account_holder: str = Field(..., description="First and last name")
        closing_balance: float
        statement_period_start: str = Field(..., description="ISO date")
        statement_period_end: str = Field(..., description="ISO date")

    for doc in documents["result"]:
        if doc.get("classification") == "bank_statement":
            extracted = client.content.extract(
                parsed_document=doc,   
                extraction_schema=BankStatement
            )
            print(extracted)
      ```

    ```bash Node
    const { z } = require("zod");

    const BankStatement = z.object({
      account_holder: z.string(),
      closing_balance: z.number(),
      statement_period_start: z.string(), // ISO date
      statement_period_end: z.string(),   // ISO date
    });

    for (const doc of documents.result) {
      if (doc.classification === "bank_statement") {
        const extracted = await client.content.extract({
          parsedDocument: doc,
          extractionSchema: BankStatement,
        });
        console.log(extracted);
      }
    }
      ```

## Custom ports

By default, Mintlify uses port 3000. You can customize the port Mintlify runs on by using the `--port` flag. For example, to run Mintlify on port 3333, use this command:

```bash
mint dev --port 3333
```

If you attempt to run Mintlify on a port that's already in use, it will use the next available port:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify versions

Please note that each CLI release is associated with a specific version of Mintlify. If your local preview does not align with the production version, please update the CLI:

```bash
npm mint update
```

## Validating links

The CLI can assist with validating links in your documentation. To identify any broken links, use the following command:

```bash
mint broken-links
```

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    This may be due to an outdated version of node. Try the following:
    1. Remove the currently-installed version of the CLI: `npm remove -g mint`
    2. Upgrade to Node v19 or higher.
    3. Reinstall the CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
  
    Solution: Go to the root of your device and delete the `~/.mintlify` folder. Then run `mint dev` again.
  </Accordion>
</AccordionGroup>

Curious about what changed in the latest CLI version? Check out the [CLI changelog](https://www.npmjs.com/package/mintlify?activeTab=versions).
